Claude Code — terminal-agent fra Anthropic

Agentic AI for utviklere som lever i terminalen. Inkludert i Claude Pro, 5,5x mer effektiv enn Cursor. Her er komplett guide.

OPPDATERT 24. APRIL 20262100+ ORDLESETID 9 MIN

Hva er Claude Code?

Claude Code er Anthropics terminal-baserte AI-agent. Der Cursor er en IDE med AI, er Claude Code en kommandolinje-agent som kan lese, skrive, endre og kjøre kode autonomt på datamaskinen din.

Hvor Cursor spør "kan jeg endre denne filen?", sier Claude Code "jeg endret disse syv filene, kjørte testene, og committet til git. Her er rapporten." Det er mer autonomt og kraftig for større oppgaver.

Hva Claude Code kan gjøre

Agentisk git-automatisering

Gi instruksjoner som kjører på tvers av mange filer og kommandoer:

$ claude
# I Claude Code-prompt:
> Klon analytics-repoen, finn hvordan User-event er definert,
  og oppdater tracking-skjemaet i dette prosjektet til å matche.
  Kjør tester etterpå og commit endringene med en god melding.

Claude Code vil cd rundt, grep for ting, endre filer, kjøre tester, og commite - alt automatisk. Du ser stdout-loggen mens den jobber.

Multi-fil refaktorering

"Endre alle forekomster av useState-hooken til useReducer i React-komponenter som har mer enn 3 state-variabler." Claude Code går gjennom prosjektet, identifiserer kandidater, gjør endringen, og kjører tester.

Automatiserte tester

"Skriv enhetstester for alle funksjoner i utils/-mappen som mangler tester." Claude Code genererer tester, kjører dem, fikser de som feiler, og iterer til alt er grønt.

VS Code-integrasjon

Claude Code har et offisielt VS Code-plugin. Du får dedikert Claude Code-panel, inline diffs, plan mode, og samtalehistorikk rett i editoren. For mange devs er "VS Code + Claude Code" den optimale workflow.

Installasjon og oppsett

Installer Claude Code via npm:

npm install -g @anthropic-ai/claude-code

# Logg inn med Claude-kontoen din
claude login

# Start i et prosjekt
cd ~/prosjekter/min-nettside
claude

Claude Code åpner en interaktiv session i terminalen. Du skriver instruksjoner i vanlig norsk eller engelsk, og den utfører dem.

Workflow-eksempler

Eksempel 1: Bygg en Next.js-app fra scratch

> Lag en Next.js 14-app i denne mappen med:
  - TypeScript og Tailwind
  - Én hovedside med hero og features-grid
  - Kontaktskjema som sender til en API-rute
  - JSON-LD schemas for SEO
  - Deploy-config for Vercel

<Claude Code jobber i 2-3 minutter>
<Sier: "Ferdig. 24 filer opprettet. Kjør 'npm run dev' for å teste.">

Eksempel 2: Fiks alle bugs i et prosjekt

> Kjør 'npm test' og fiks alle feil til alle tester passerer.
  Hvis noen tester er dårlig skrevet, oppdater dem også.

<Claude Code kjører tester, identifiserer 7 feil,
  fikser dem en etter en, kjører tester igjen>
<Gjentar til alle tester er grønne>

Eksempel 3: Oppdater avhengigheter trygt

> Oppdater alle npm-pakker til nyeste versjon.
  Etter hver oppdatering, kjør tester og lint.
  Hvis noe brekker, revert og noter det i en CHANGELOG.md.

<Claude Code oppdaterer 34 pakker systematisk,
  reverterer 2 som brekker, dokumenterer hvorfor>

Claude Code vs Cursor: ærlig vurdering

Velg Claude Code hvis:

  • Du lever allerede i terminalen (tmux, fzf, vim)
  • Du jobber med DevOps eller backend på mange mikrotjenester
  • Du vil ha autonome multi-steg-oppgaver
  • Du har Claude Pro fra før (inkludert)
  • Du foretrekker agentic workflow fremfor inline-kontroll

Velg Cursor hvis:

  • Du foretrekker GUI og visuelle diffs
  • Du vil se hver endring før den skjer
  • Du er vant til VS Code-workflow
  • Tab-to-complete er hovedbehovet ditt

Beste kompromiss: Bruk Claude Code i VS Code via det offisielle plugin. Du får autonomi når du vil ha det, og visuell kontroll når du trenger det.
